```python
def missing_number(nums: list[int]) -> int:
    n = len(nums) + 1
    expected = n * (n + 1) // 2
    return expected - sum(nums)
```
### Оценка по времени 
Происходит один проход по последовательности — $O(n)$. Для неотсортированного массива это оптимально: не прочитав какой-то элемент, нельзя быть уверенным, что пропущено не его значение, поэтому $\Omega(n)$ — нижняя граница. Бинарный поиск даёт $O(\log n)$, но только за счёт дополнительного предположения об упорядоченности.

### Оценка по памяти
Используется лишь несколько переменных — $O(1)$ дополнительной памяти.

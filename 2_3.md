```python
def prime_factors(n: int) -> list[int]:
    factors = []

    while n % 2 == 0:
        factors.append(2)
        n //= 2

    d = 3
    while d * d <= n:
        while n % d == 0:
            factors.append(d)
            n //= d
        d += 2

    if n > 1:
        factors.append(n)

    return factors
```
### Оценка по времени 
В худшем случае (когда $n$ простое или имеет большой простой делитель) внешний цикл проходит нечётные значения от 3 до $\sqrt{n}$, то есть около $\sqrt{n} / 2$ итераций — итого $O(\sqrt{n})$. 

### Оценка по памяти
Дополнительной памяти, кроме результата, нужно $O(1)$ за счёт переменных. Сам список множителей содержит не более $\log_2 n$ элементов: каждый множитель не меньше 2, а их произведение равно $n$. Итого: $O(\log n)$.
